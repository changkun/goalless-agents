# SmartCommit Examples

This document shows real-world examples of commit messages generated by SmartCommit.

## Example 1: Adding a New Feature

**Changes:**
```python
# api.py - New file
def create_user(username, email):
    """Create a new user account"""
    user = User(username=username, email=email)
    db.session.add(user)
    db.session.commit()
    return user
```

**Generated Message:**
```
feat(api): add api implementation

- Modify api.py
- Add 145 lines
```

## Tips for Best Results

1. **Stage Related Changes**: Group related changes together for better commit messages
2. **Small Commits**: Smaller, focused commits generate more accurate messages
3. **Review Generated Messages**: Always review before committing
4. **Use Interactive Mode**: Edit messages when you need more context

## Customizing Messages

You can enhance SmartCommit's analysis by:

- Adding code comments that explain the "why"
- Using descriptive variable and function names
- Following consistent file naming conventions
- Organizing code into logical modules

SmartCommit uses these patterns to generate more accurate and meaningful commit messages.
